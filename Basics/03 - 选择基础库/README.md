> 如果你要制作一个人物mod，选择一个基础库可以减少你的patch数量并减轻你的工作。同时也可以提高和其他mod的兼容性。

目前（工坊未开放之前）常用的基础库有[BaseLib](https://github.com/Alchyr/BaseLib-StS2)和[RitsuLib](https://github.com/BAKAOLC/STS2-RitsuLib)两个。

## 对比

以下表格为这些基础库目前为止（2026.04.18）支持的功能表格。其中“添加xxx”的功能意思是游戏虽支持，但是该基础库对其做了优化。

| 功能 | BaseLib | RitsuLib | 说明 |
|-----------|------|------|------|
| 添加配置选项 | ✅ | ✅ | RitsuLib功能更丰富，支持镜像用BaseLib开发的mod的配置 |
| 自动注册 | ✅ | ✅ | - |
| 自定义素材路径 | ✅ | ✅ | - |
| 角色素材缺少时使用原版 | ✅ | ✅ | - |
| dynamicvar的提示文本 | ✅ | ✅ | - |
| 添加人物 | ✅ | ✅ | - |
| 关键词注册 | ✅ | ✅ | - |
| customenum | ✅ | ❌ | baselib支持对任意enum扩展，目前不稳定仍在开发阶段；ritsulib不支持，但统一管理（例如关键词） |
| 人物非spine动画兼容 | ✅ | ❌ | baselib会自动调用非spine动画的动画名 |
| 不挂载脚本场景转换 | ✅ | ❌ | baselib会转换 |
| FMOD音频 | 🚧 | ✅ | - |
| 缺少资源时占位警告 | ❌ | ✅ | - |
| 时间线注册 | ❌ | ✅ | 游戏时间线小故事 |
| 事件管线 | ❌ | ✅ | 订阅事件，触发时调用 |
| 内容物ID | 命名空间第一段大写，例如`TEST-TEST_CARD` | modid，例如`TEST_TEST_CARD` | - |
| patch | 原始`harmony` | 原始`harmony`以及封装patch系统 | - |
| 非原生玩法内容 | 有 | 无 | 避免重复造轮子。但是会带来臃肿不兼容、抢占patch、抢占你的命名的问题。 |

## 重要说明

* 如果你不添加任何游戏内容，不需要基础库也可以。但是如果你制作人物mod等，最好使用一个基础库减少工作量并提高兼容性。

* 目前baselib的教程已相对完善，但是baselib开发相对缓慢（~~并且固执己见地添加了许多意义不明的功能~~）。

* ritsulib的教程正在编写中。
